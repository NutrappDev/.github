# Política de Seguridad

## Versiones con soporte activo

Solo la última versión en `main` de cada repositorio recibe parches de seguridad.

## Reportar una vulnerabilidad

**No abrir un issue público.** Las vulnerabilidades de seguridad deben reportarse de forma privada.

### Cómo reportar

1. Usar [GitHub Private Security Advisories](https://github.com/NutrappDev/.github/security/advisories/new) en el repo afectado
2. O enviar un correo a: **<desarrollo@nutrapp.com.co>**

Incluir en el reporte:

- Descripción del problema y su impacto potencial
- Pasos para reproducirlo
- Versión/rama afectada
- Cualquier mitigación conocida

### Tiempos de respuesta

| Acción                    | Plazo            |
| ------------------------- | ---------------- |
| Confirmación de recepción | 48 horas hábiles |
| Evaluación de severidad   | 10 días hábiles  |
| Plan de mitigación        | 20 días hábiles  |

### Lo que NO hacer

- No publicar detalles de la vulnerabilidad antes de que exista un parche
- No explotar la vulnerabilidad más allá de lo necesario para demostrarla
- No acceder a datos de otros usuarios

Agradecemos la divulgación responsable y reconoceremos públicamente a los contribuidores que sigan este proceso.
